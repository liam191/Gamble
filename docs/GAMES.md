# Game Rules & Logic

## 공통 규칙

- **배팅 통화**: ETH, WETH, UP, SIDE, SEC, USP
- **당첨금**: UP 토큰으로 지급 (오라클 가격 기준 환산)
- **하우스 엣지**: 4%
- **최소 배팅**: ~$20 (0.01 ETH)
- **최대 배팅**: ~$200 (0.1 ETH)
- **배당 공식**: `배팅액 × 0.96 × (modulo / 선택한 경우의 수)`

## Commit-Reveal 구조와 프론트엔드 연동

모든 게임은 동일한 commit-reveal 프로토콜을 사용합니다. 프론트엔드에서의 흐름:

```
[게임 선택 + 배팅 설정]
        │
        ▼
[Place Bet 클릭] ── POST /api/commit ──> 백엔드가 secret 생성
        │                                 commit = keccak256(secret)
        │           <── {commit, signature, commitLastBlock} ──
        ▼
[지갑 서명 팝업] ── placeBetETH(commit, signature, ...) ──> 온체인 기록
        │                                                   blockhash 확정
        ▼
[결과 대기 UI]   <── 백엔드가 BetPlaced 감지 ──> settleBet(secret) TX
        │              keccak256(secret, blockhash) % modulo = 결과
        ▼
[결과 표시]      <── BetSettled 이벤트 폴링 ──> diceResult, payout 수신
```

**프론트엔드는 결과를 결정하지 않습니다.** 프론트엔드의 역할:
1. `/api/commit`으로 commit + 서명을 받아옴
2. 유저의 지갑으로 `placeBetETH` TX를 전송
3. `BetSettled` 이벤트를 폴링하여 `diceResult`를 수신
4. `diceResult`에 따라 게임별 연출(카드 딜링, 룰렛 회전 등)을 결정론적으로 생성

카드 게임(Dragon Tiger, Baccarat)의 카드 연출도 `diceResult`를 시드로 결정론적으로 생성되므로, 같은 결과에 대해 항상 같은 카드가 표시됩니다. 자세한 공정성 분석은 [Fairness & Security](FAIRNESS.md)를 참고하세요.

## 1. Coin Flip (홀짝)

```
modulo = 2
결과: 0 (Heads) 또는 1 (Tails)
```

| 선택 | betMask | 확률 | 배당 |
|------|---------|------|------|
| Heads | 1 | 50% | 1.92x |
| Tails | 2 | 50% | 1.92x |

**플레이 방법**: Heads 또는 Tails 중 하나를 선택하고 배팅합니다.

---

## 2. Dice (주사위)

```
modulo = 6
결과: 0~5 (표시: 1~6)
```

| 선택 | betMask | 확률 | 배당 |
|------|---------|------|------|
| 숫자 1개 | 2^n | 16.7% | 5.76x |
| 숫자 2개 | 합 | 33.3% | 2.88x |
| 숫자 3개 | 합 | 50% | 1.92x |
| ... | ... | ... | ... |

**플레이 방법**: 1~6 중 하나 이상의 숫자를 선택합니다. 복수 선택 가능. 선택한 숫자 중 하나가 나오면 승리합니다. 많이 선택할수록 확률은 높아지지만 배당은 낮아집니다.

**비트마스크 인코딩**:
```
숫자 1 = bit 0 = betMask 1
숫자 2 = bit 1 = betMask 2
숫자 3 = bit 2 = betMask 4
...
숫자 1,3,5 선택 = 1 + 4 + 16 = betMask 21
```

## 3. Double Dice (더블 다이스)

```
modulo = 36
결과: 0~35 → 주사위1 = result/6, 주사위2 = result%6
합계: 2~12
```

| 합계 | 경우의 수 | 확률 | 배당 |
|------|----------|------|------|
| 2 | 1/36 | 2.8% | 34.56x |
| 3 | 2/36 | 5.6% | 17.28x |
| 4 | 3/36 | 8.3% | 11.52x |
| 5 | 4/36 | 11.1% | 8.64x |
| 6 | 5/36 | 13.9% | 6.91x |
| 7 | 6/36 | 16.7% | 5.76x |
| 8 | 5/36 | 13.9% | 6.91x |
| 9 | 4/36 | 11.1% | 8.64x |
| 10 | 3/36 | 8.3% | 11.52x |
| 11 | 2/36 | 5.6% | 17.28x |
| 12 | 1/36 | 2.8% | 34.56x |

**플레이 방법**: 두 주사위의 합계(2~12)를 선택합니다. 복수 선택 가능. 선택한 합계가 나오면 승리합니다.

## 4. Roulette (룰렛)

```
modulo = 37
결과: 0~36 (유럽식 룰렛)
```

**배팅 종류**:

| 배팅 | 선택 수 | 확률 | 배당 |
|------|---------|------|------|
| 단일 숫자 | 1/37 | 2.7% | 35.56x |
| Red / Black | 18/37 | 48.6% | 1.97x |
| Odd / Even | 18/37 | 48.6% | 1.97x |
| 1-18 / 19-36 | 18/37 | 48.6% | 1.97x |
| 1st/2nd/3rd Dozen | 12/37 | 32.4% | 2.96x |
| Column | 12/37 | 32.4% | 2.96x |

**숫자 배색** (유럽식 표준):
```
Green: 0
Red:   1, 3, 5, 7, 9, 12, 14, 16, 18, 19, 21, 23, 25, 27, 30, 32, 34, 36
Black: 2, 4, 6, 8, 10, 11, 13, 15, 17, 20, 22, 24, 26, 28, 29, 31, 33, 35
```

**휠 순서** (유럽식 표준):
```
0-32-15-19-4-21-2-25-17-34-6-27-13-36-11-30-8-23-10-
5-24-16-33-1-20-14-31-9-22-18-29-7-28-12-35-3-26
```

**플레이 방법**: 배팅 테이블에서 개별 숫자를 클릭하거나, 하단의 프리셋(Red, Black, Odd, Even 등)을 선택합니다. 복수 선택 가능.

## 5. Dragon Tiger (드래곤 타이거)

```
modulo = 13
결과: 0 = Tie, 1~6 = Dragon, 7~12 = Tiger
```

| 배팅 | 확률 | 배당 |
|------|------|------|
| Dragon | 46.2% (6/13) | 2.08x |
| Tiger | 46.2% (6/13) | 2.08x |
| Tie | 7.7% (1/13) | 12.48x |

**규칙**: Dragon과 Tiger에 각각 카드 1장이 딜링됩니다. 높은 카드를 가진 쪽이 승리합니다. 같으면 Tie입니다.

**카드 연출**: 결과가 확정된 후, commit hash를 시드로 결정론적으로 카드쌍이 생성됩니다. 카드는 순차적으로 뒤집히며 공개됩니다.

**플레이 방법**: Dragon, Tiger, Tie 중 하나를 선택하고 배팅합니다.

## 6. Baccarat (바카라)

```
modulo = 13
결과: 0 = Tie, 1~6 = Player, 7~12 = Banker
```

| 배팅 | 확률 | 배당 |
|------|------|------|
| Player | 46.2% (6/13) | 2.08x |
| Banker | 46.2% (6/13) | 2.08x |
| Tie | 7.7% (1/13) | 12.48x |

**카드 점수 계산**:
```
Ace = 1점
2~9 = 숫자 그대로
10, J, Q, K = 0점
합계의 일의 자리만 사용 (예: 7+8=15 → 5점)
```

**정식 바카라 규칙이 적용됩니다**:

1. Player와 Banker에 각각 2장씩 딜링
2. **Natural**: 어느 한쪽이 8 또는 9이면 즉시 승부 결정
3. **Player 3번째 카드**: 합계 0~5면 1장 추가, 6~7이면 스탠드
4. **Banker 3번째 카드** (Player가 3번째를 받은 경우):

| Banker 합계 | Player 3번째 카드 값이 이것이면 Banker도 받음 |
|------------|----------------------------------------------|
| 0~2 | 무조건 받음 |
| 3 | 8이 아니면 받음 |
| 4 | 2~7이면 받음 |
| 5 | 4~7이면 받음 |
| 6 | 6~7이면 받음 |
| 7 | 스탠드 |

Player가 스탠드(3번째 카드 없음)인 경우: Banker 0~5면 받음, 6~7 스탠드.

**카드 연출**: 결과 확정 후 commit hash 시드로 카드를 생성하되, 바카라 정식 룰을 따르는 카드 조합을 찾을 때까지 시드를 반복 시도합니다. 카드는 P1→B1→P2→B2 순서로 공개되며, 초기 합계 표시 후 3번째 카드가 있으면 순차 공개됩니다. 결과 배너는 모든 카드 공개 후에만 표시됩니다.

**플레이 방법**: Player, Banker, Tie 중 하나를 선택하고 배팅합니다.

## 7. Hi-Lo (하이로)

```
modulo = 100
결과: 0~99
승리 조건: 결과 < rollUnder (선택한 값)
```

| 설정 | 승률 | 배당 |
|------|------|------|
| rollUnder = 50 | 50% | 1.92x |
| rollUnder = 75 | 75% | 1.28x |
| rollUnder = 25 | 25% | 3.84x |
| rollUnder = 10 | 10% | 9.60x |
| rollUnder = 90 | 90% | 1.07x |

**범위**: 5~95 (하우스 리스크 제한)

**플레이 방법**: 슬라이더를 움직여 승률을 5%~95% 사이에서 자유롭게 설정합니다. 왼쪽(낮은 승률)으로 갈수록 배당이 높고, 오른쪽(높은 승률)으로 갈수록 배당이 낮습니다. 0~99 중 랜덤 숫자가 나와서 설정한 값보다 작으면 승리합니다.
